BFS vs DFS

Two concepts.

  1. Visiting a Vertex
  2. Exploration of Vertex

BFS - Breadth First Search DFS - Depth First Search

BFS - Visit any start node, explore each adjacent node first, then visit next node.. iterate DFS - Visit any start node, explore one adjacent node at a time, till end leaf, then visit next node.. iterate

BFS - Checks level by level DFS - Goes to deepest child of a node, then explore next. Its branch by branch, not level by level.

BFS - At the end we get, BFS Spanning Tree. Edges are called cross edges. DFS - At the end we get, DFS Spanning Tree. Edges are called back edges.

BFS - We don't need a DS to run this DFS - We need to use stack DS to keep track of ancestor nodes.

https://youtu.be/pcKY4hjDrxk?list=PLDN4rrl48XKpZkf03iYFl-O29szjTrs_O&t=735 In DFS, a stack is used to keep track of the parent nodes. Since we keep going deep, we must know how to go back, hence stack keeps track of all ancestor nodes of the given visited node at all stages. When reached the leaf, and going back up, stack is one by one made empty.
